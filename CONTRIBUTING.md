# Contributing

Contributions, issue reports, and design-review feedback are welcome.

Before submitting a change:

1. Describe the hardware or firmware change clearly.
2. Include the affected schematic sheets, PCB areas, or source files.
3. Explain how the change was tested.
4. Do not commit credentials, licensed component libraries, or proprietary files.
5. For hardware revisions, include updated fabrication outputs only after design review.
